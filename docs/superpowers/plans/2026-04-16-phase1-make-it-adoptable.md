# Phase 1: "Make It Adoptable" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Borg installable by anyone (including non-developers) in under 5 minutes, and connectable to any project with one command — so a second Rackspace team can adopt it with zero help.

**Architecture:** A `borg` CLI (Python, installed via pipx or the installer script) handles installation, project init, and bulk ingestion. The existing FastAPI engine and Postgres remain untouched. A new `/api/admin/cost-summary` endpoint aggregates extraction token costs. Agent-hook templates are static files generated by `borg init`.

**Tech Stack:** Python 3.12+, Click (CLI framework), Jinja2 (templates), Docker Compose, shell scripting (installer), existing FastAPI + asyncpg stack (cost endpoint)
